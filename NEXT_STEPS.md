## 📞 How to Talk About This in Interviews

### "Tell me about a project you're proud of"

```
"I built a data consolidation project that unified 7 independent support 
programs into one queryable source of truth. 

The business problem: Each program had its own data structure, metrics, 
and team. Leadership couldn't see cross-program trends, and the support 
ops team had to ask 7 different people for data.

My approach:
1. Built a dbt pipeline with proper layering (raw → staging → mart)
2. Handled schema heterogeneity (different column names and types)
3. Managed semantic heterogeneity (status values: IN_PROGRESS vs in_progress vs wip)
4. Solved timezone complexity (UTC conversion for global SLA metrics)
5. Pre-computed metrics for performance
6. Added Claude Desktop integration so teams can query in plain English

Result: 7,000 unified tickets with zero data quality issues. Non-technical 
teams can now self-serve analytics. The data is AI-ready.

Technologies: dbt, SQL, DuckDB, Python (MCP protocol)"
```

### "What would you do differently?"

```
"In production, I'd probably:
1. Add incremental loading (only new tickets)
2. Implement dbt tests (not_null, unique, referential integrity)
3. Set up automated runs (Airflow, dbt Cloud, or Goose Desktop)
4. Add more granular metrics (by day, by issue, by resolution path)
5. Create Looker/Tableau dashboards on top of the metrics
6. Add data lineage tracking (dbt metadata)
7. Set up data quality monitoring (Great Expectations)"
```
